# 管理面板

运行一行命令，在浏览器里查看邮件桥的实时运行状态。

## 启动

```bash
agently-mail dashboard
# 默认监听 http://localhost:3030
# 可通过 --port 指定端口
agently-mail dashboard --port 8080
```

浏览器自动打开管理面板。

## 面板内容

### 概览

- 当前运行状态（是否在轮询）
- 最近一次 poll 时间
- 今日已处理 / 已拦截邮件数

### 访问控制

| 区域 | 内容 |
|------|------|
| 静态白名单 | 来自 `email-acl.yaml` 的 `allowed_senders` |
| 静态黑名单 | 来自 `email-acl.yaml` 的 `denied_senders` |
| 动态白名单 | 通过管理员指令 `/allow` 添加的地址 |
| 动态黑名单 | 通过管理员指令 `/deny` 添加的地址 |
| 即时回复名单 | `instant_reply_senders` 中的地址 |

### 批处理队列

批处理模式启用时显示：

- 当前队列中等待处理的邮件列表
- 每封邮件的发件人、主题、进入时间、正文摘要
- 下次摘要邮件发送倒计时

### 处理历史

最近 100 条处理记录：

- 邮件发件人、主题、处理时间
- 处理结果（已回复 / 已跳过 / ACL 拦截 / 失败）
- 使用的 Profile 名称

### 配置状态

当前加载的配置快照：

- `email-profiles.yaml` 中所有 Profile 及其触发器
- `email-acl.yaml` 中的关键配置项
- Poll 间隔、批处理间隔等运行参数

## 安全说明

Dashboard 仅监听本地地址（`127.0.0.1`），不对外网暴露。面板是**只读**的——所有管理操作仍需通过管理员邮件指令进行。
